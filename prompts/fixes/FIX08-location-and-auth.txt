In users/views.py add LocationUpdateView:
class LocationUpdateView(APIView):
    permission_classes = [IsAuthenticated]
    def post(self, request):
        lat = request.data.get('lat')
        lng = request.data.get('lng')
        if lat and lng:
            request.user.lat = float(lat)
            request.user.lng = float(lng)
            # Reverse geocode to update village/district
            # Just save lat/lng — frontend handles display
            request.user.save(update_fields=['lat', 'lng'])
        return Response({'status': 'location updated'})

Add to users/urls.py:
  path('location/', LocationUpdateView.as_view()),

--- FRONTEND: Login location update ---

src/pages/LoginPage.jsx:
After successful login (auth token received):
  Silently call geolocation in background (do NOT await, do NOT block UI):
  
  navigator.geolocation.getCurrentPosition(
    (pos) => {
      usersAPI.updateLocation({
        lat: pos.coords.latitude,
        lng: pos.coords.longitude
      }).catch(() => {}) // silently ignore errors
    },
    () => {} // silently ignore if denied
  )
  
  Then immediately navigate to dashboard — don't wait for location.
